## 2. MSIL and CLR (Very Important â€“ Asked in 2080, 2081-II)

### MSIL (Microsoft Intermediate Language)
- Also called CIL (Common Intermediate Language)
- CPU-independent intermediate code generated by .NET compilers
- Contains instructions for loading, storing, calling methods, exception handling
- Stored in assemblies (.dll, .exe) along with metadata[1][2]

### CLR (Common Language Runtime)
- Execution engine of .NET Framework
- Manages code execution (managed code)
- **Key services provided by CLR:**
  - **JIT Compilation**: Converts MSIL to native machine code at runtime
  - **Garbage Collection**: Automatic memory management
  - **Exception Handling**: Handles runtime errors
  - **Type Safety**: Ensures correct use of data types
  - **Security**: Code access security, verification
  - **Language Interoperability**: Different .NET languages work together[3][4]

### Compilation Process
```
Source Code (C#, VB) â†’ Compiler â†’ MSIL + Metadata (Assembly)
                                        â†“
                          CLR (JIT) â†’ Native Machine Code â†’ Execution
```

## 2. MVC Pattern (Very Important â€“ Asked in 2081)

**Model-View-Controller** separates application into three components:

- **Model**: Data and business logic (e.g., `Student`, `Book` classes)
- **View**: User interface (Razor views `.cshtml`)

### MVC Flow
```
User Request â†’ Controller â†’ Model (data) â†’ View â†’ Response to User
```

### Example Answer: "Explain MVC Pattern in ASP.NET" [7 marks]

The MVC (Model-View-Controller) pattern divides an application into three interconnected components:

**Model:**
- Represents data and business logic
- Contains classes like `Student`, `Book` with properties
- Handles data validation and database operations

**View:**
- Displays data to the user (UI layer)
- In ASP.NET Core, views are `.cshtml` files using Razor syntax
- Should not contain business logic

**Controller:**
- Handles incoming HTTP requests
- Interacts with Model to get/update data
- Selects appropriate View to render response

**Advantages of MVC:**
- Separation of concerns (easy maintenance)
- Testability (each component can be tested independently)
- Clean HTML output (full control over markup)
- Supports multiple views for same data

## 4. Dependency Injection and IOC Container (Very Important â€“ Asked in 2081-II)

### What is Dependency Injection (DI)?
DI is a design pattern where dependencies are provided to a class rather than created inside it. This promotes loose coupling and testability.[12][13]

### What is IOC Container?
**Inversion of Control (IOC) Container** is a framework that manages object creation and dependency injection automatically.[14]

### ASP.NET Core Built-in DI Container
ASP.NET Core has built-in DI support using `IServiceCollection` and `IServiceProvider`.

**Steps:**
1. **Register** services in `Program.cs`
2. **Resolve** (inject) via constructor

### Example Code

**Step 1: Create Interface and Implementation**
```csharp
public interface IStudentService
{
    List<Student> GetAllStudents();
}

public class StudentService : IStudentService
{
    public List<Student> GetAllStudents()
    {
        return new List<Student>
        {
            new Student { Id = 1, Name = "Ram" },
            new Student { Id = 2, Name = "Shyam" }
        };
    }
}
```

**Step 2: Register Service in Program.cs**
```csharp
var builder = WebApplication.CreateBuilder(args);

// Register service with DI container
builder.Services.AddScoped<IStudentService, StudentService>();

builder.Services.AddControllersWithViews();
var app = builder.Build();
```

**Step 3: Inject in Controller (Constructor Injection)**
```csharp
public class StudentController : Controller
{
    private readonly IStudentService _studentService;

    // Constructor injection
    public StudentController(IStudentService studentService)
    {
        _studentService = studentService;
    }

    public IActionResult Index()
    {
        var students = _studentService.GetAllStudents();
        return View(students);
    }
}
```

### Service Lifetimes

| Lifetime | Description |
|----------|-------------|
| `AddSingleton` | Single instance for entire application |
| `AddScoped` | One instance per HTTP request |
| `AddTransient` | New instance every time requested |
